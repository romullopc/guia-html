HEAD

Não possui tag's visiveis, apenas tag de configurações do documento HTML.


!DOCTYPE-html --- Instrução para o navegador que diz qual versão, ou, qual documento do HTML estamos trabalhando

base --- primeira meta tag que indica onde esta armazenado conteúdos de hyperlinks do documento html, sendo referência root caso utilize uma tag de hyperlinks.

atributos base

href="" --- atributo obrigatório que indica o caminho dos hyperlinks

A meta tag base é um facilitador de todos hyperlinks do HTML, quando invocado no documento, deve-se ocultar o nome diretório onde esta armezenado o documento referido no hyperlink.

Caso adicione hyperlink que se refira a outro conteúdo, atente-se de voltar o diretório com o atalho "../" e oriente o caminho até o diretório desejado.

target --- indica onde e como será acessado o hyperlink

html lang="pt-br" Lang é um ?parâmetro? da tag html (tag root) que indica qual idioma do documento html.

title --- Título do documento html, pode conferir na barra do navegador.

Tipos de atributos

boolean --- atributo com duas possibilidades, caso TRUE o atributo é ativado; caso FALSE o atributo é omitido

submit --- atributo para elemento do tipo de enviar dados.

tags semânticas

article --- é um elemento semântico que representa um conteúdo independente, que pode ser distribuído ou reutilizado de forma isolada. Ela é ideal para marcar conteúdos como artigos de revista, postagens de blog, notícias ou qualquer outra seção de conteúdo autocontida; Utilizada opcionalmente quando o conteúdo seja diferente de "header, nav, footer"

Principais características da tag article:

Conteúdo Autônomo: O conteúdo dentro de uma tag article faz sentido sozinho, sem depender de outros elementos.

Reutilizável: Pode ser compartilhado ou republicado independentemente, como em feeds RSS.

Exemplos de uso: Postagens de blog, artigos em jornais, revistas, comentários ou respostas em um fórum.

aside --- tag semântica, é um conteúdo secundário deve estar diretamente relacionado ao conteúdo circundante. geralmente usado para publicidade, nota de curiosidades, citação de um artigo ou uma lista de link(nav), "blogroll".

atributos âncora

href="link" --- indica qual caminho a seguir, se o caminho for um #id, poderá navegar para parte específica na propria pagina web.

Observar ao alterar os arquivos de diretório, pois o caminho de todos os links há de ser redefinidos.

Target="x" --- indica em qual janela o video ligado a tag âncora será direcionado.

rel="x" --- indica qual tipo de arquivo será chamado.

media="x" --- em qual tipo de media será impresso.

hreflang="x" --- indica qual tipo de lingua esta o caminho chamado.

type="x" --- tipo de conteúdo codificado esta a página MIME types.

download="x" orienta o browser para que facilitar a identificar no desenvolvimento o tipo de arquivo.

audio ---tag que reproduz audio no documento html

atributos audio

src --- atributo que direciona o caminho onde esta armazenado o arquivo

autoplay --- atributo (boolean) que inicializa o audio junto ao carregamento da página HTML.

mute --- atributo (boolean) que silencia o audio junto ao carregamento da página HTML.

controls --- atributo (boolean) que disponibiliza o controle de audio ao usuário

loop --- atributo (boolean) reinicializa o audio toda vez que tremina.

abbr --- usada para definir uma abreviação ou sigla. Quando você passa o cursor sobre o texto, aparece a forma completa da abreviação, exemplo: HTML

address --- utilizada para definir informações de contato ou endereço. Normalmente, é usada para identificar o autor de um documento ou artigo, incluindo informações como endereço, e-mail e telefone.

Um documento HTML pode conter múltiplas tags address. No entanto, cada tag address deve ser usada para representar um único bloco de informações de contato, como o endereço do autor de um documento, o endereço de uma empresa, etc. Não há um limite estrito sobre quantas tags address podem ser usadas, mas é importante garantir que cada uma seja utilizada de maneira semântica e clara.

b --- tag contida em um fluxo de frase cujo o objetivo é destacar como negrito; não possui atributos.

bdo --- tag que indica o fluxo que a frase é escrita, com o atributo DIR possibilitamos a dirção (ltr ou rtl).

blockquote --- o elemento representa citação de ountra fonte, contendo elementos fraseados e geralmente no rodapé citando os autores e a obra. (destacar o blockquote)

br --- O elemento br representa uma quebra de linha. Os elementos br devem ser utilizados apenas para quebras de linha que realmente façam parte do conteúdo, como em poemas ou endereços. Os elementos br não devem ser utilizados para separar grupos temáticos em um parágrafo. Usar apenas um br por linha!

cite --- citação curta que pode informar o autor ou uma obra externa ao documento, geralmente apresentada no rodapé.

map --- mapear e cirar area clicavel, contem um atributo name para identificar a imagem.

area --- contido na tag map, a tag area serve para definir locais 'clicaveis' dentro de um poligno desejado, geralmente usado em imagens.

atributos area

alt --- frase alternativa caso não carregue a imagem.

shape --- forma de mapear a area, seja 'circle', 'rect' ou 'poly'.

href="#" --- caminho clicavel para area mapeada.

coords=" " --- coordenadas de mapeamento da imagem, no Paint, encontre as coordenadas e separe-as.

button --- adiciona um botão no documento, pode ser contido em um fraseamento e ser fraseado.

tipos de button

submit --- botão de enviar dados

reset --- botão de limpar dados

button --- botão genérico

atributos button

autofocus --- Atributo booleano que ao carregar a página web o elemento ja está focado para enviar dados.

disable --- Atributo booleano que desabilita o botão.

form --- associa o botão a um formulário por ID.

formaction --- ao enviar os dados associa o botão a uma URL.

formenctype --- atributo para enviar arquivos para o beckend (PHP).

formmethod --- protocolo HTTPque sera usado para o tipos de formulários, se é post ou get.

formvalidate --- validação de formulário.

formtarget --- local onde vai abrir o formulário.

name --- nome do botão.

value --- qual formato da validação do botão, radio, checkbox...

canvas --- no HTML é usada para desenhar gráficos, animações e outras imagens através de scripts, geralmente JavaScript. Ela cria uma área de desenho em uma página web onde você pode usar métodos e propriedades para desenhar formas, gráficos, e até mesmo criar jogos.

atributos canvas

height --- define a area vertical do quadro.

width --- define a area horizontal do quadro.

id --- forma de atribuir propriedades JavaScript ao id do canvas desejado.

É necessario utilizar JavaScript para manipular propriedades do canvas

code --- usado para definir um trecho de código de programação. Normalmente, ele é exibido em uma fonte monoespaçada, o que significa que cada caractere ocupa a mesma quantidade de espaço horizontal.

A tag code é contido dentro da tag pre, para manter a formatação do código e respeitando as condições de hieraquia dos elementos parent.

pre --- em HTML significa "preformatted text", ou seja, texto pré-formatado. Quando você usa essa tag, o texto dentro dela é exibido exatamente como foi escrito, preservando todos os espaços em branco, quebras de linha e a formatação.

del --- utilizada para marcar texto que foi deletado ou removido de um documento. Normalmente, o texto dentro dessa tag é exibido com um traço no meio, indicando que ele foi excluído ou está obsoleto. exemplo

ins --- usada para marcar texto que foi inserido ou adicionado a um documento. Assim como a tag del, ela é frequentemente usada para mostrar revisões ou mudanças em um texto. exemplo

atributos das tags del e ins

cite="" --- hyperLink para a fonte da citação ou mais informações sobre aa tag inserida.

datetime="23-03-2025T22:36:00" --- informa o dia e horario da inserção das tags del e ins do documento.

details --- usado para criar uma seção recolhível que os usuários podem expandir ou ocultar, geralmente para exibir informações adicionais. Ele costuma ser combinado com a tag para definir o cabeçalho visível da seção recolhível.

atributos details

details é um elemento booleano, ou seja, possui apenas um atributo de interação: open

geralmento utilizado em FAQs, complemento de explicações de atendimento ao cliente, ocultando conteúdo que não precisa ser visível imediatamente.

summary --- título atribuído ao details integrado.

há possibilidades de formatar a seta do summary com propriedades css.

dfn --- destaca abreviações ou textos definidos. (Geralmente é usado na primeira ocorrência de um termo que será explicado ou definido no mesmo contexto).

diferente do abbr que destaca abreviações ou acrônimos, ex.: NASA, HTML, BRICS...

Formulário

fieldset --- tag de agrupamento de elementos de um formulário; aceita o atributo boolean disabled e legend.

disabled desabilita o grupo de elementos fieldset; com JavaScript possibilitamos disponibilizar como a próxima etapa do formulário.

o atributo Legend é filho direto de fieldset.

textarea --- área de inserção de texto no formulário disponibilizando atributos cols e rows.

aceita apneas entrada de textos, não possibilita tags html.

possibilita atributos autofocus e maxlength.

atributos

action --- URL receberá o envio dos dados do formulário.

autocomplete --- autocompletar campos ativados.

enctype --- codificação (tipo) dos dados enviados.

method --- método de envio de dados (get ou post).

name --- nome relacionado ao formulário.

novalidade --- desabilita a validação pelo browser.

target --- como a página de destino (action) será aberta.

input atributos

name --- nome identificação do input.

value --- valor demonstrado no input.

maxlength --- quantidade de carcteres no valor do input (quantidade de letras).

size --- quantidade de visibilidade de caracteres no valor do input. (tamanho da barra).

required --- atributo tipo boolean, define o input como campo obrigatório do formulário.

pattern --- Formatação da caixa de input para os dados serer inseridos de forma desejada.

placeholder --- informar campo obrigatório de dados do input.

step --- passo de contagem de um número a outro

type --- tipo de imput a ser trabalhado.

tipos de input

checkbox --- caixa de verificação.

quando criar este tipo de input, facilitar ao maximo a leitura para o banco de dados

criar "name" diferentes, pois quando analizar a verificação no server side, facilitará a leitura.

radio --- radio de seleção único elemento.

diferente do checkbox, input radio deve conter o mesmo name de identificação. desta forma não há como selecionar mais de um radio.

adicionar o atributo checked para o raidio ja inicializar com marcação. (checked é um atributo booleano)

time --- Tipo de input que trabalha com horas.

este tipo de input possibilita receber o atributos 'min' e 'max', definindo o horário minimo e maximo no input.

Date --- tipo de input que possibilita a escolha de dia, mês e ano; possibilita os atributos 'min' e 'max'

datetime-local --- tipo de input que trabalha com a data e hora na mesma caixa input.

month --- tipo de input que disponibiliza apenas a caixa que retorna o mês.

week --- tipo de input que desponibiliza uma caixa que ajuda a retornar a semana do mês.

hidden --- input "invisivel" com o objetivo de enviar arquivos para o formulário sem interação com usuário

color --- caixa de input que possibilita a escolha de uma cor, e retorna um valor hexadecimal.

email --- caixa de input que recebe email. (apenas será tratado no banco de dados)

image --- input image permite e inserção de uma imagem tipo botão no formulário. aceita atributo "src"

tell --- caixa input tipo texto para inserir o numero de telefone.

boas práticas inserir o atributo pattern e placeholder para formatar como deve ser inserido o texto

number --- caixa input tipo números; aceita atributos "setp", "min" e "max"

search --- caixa de input tipo texto próprio para entrada de "string" para pesquisa.

url --- caixa de input tipo texto que envia uma url.

boas práticas inserir o placeholder para informar o formato aceito pelo formulário;

inserir o atributo pattern para formatar a validação do campo de texto da url.

file --- caixa input para inserção de arquivos no formulario

definir o enctype na tag "form" para compreender o tipo de arquivo que será trabalhado em tal formulário;

há possibilidade de inserir o atributo "multiple"(boolean) para enviar mais de um arquivo por vez;

definir o atributo accept para filtar o tipo de arquivo que o input irá trabalhar.

tipos de accept file

accept=".doc,.docx" --- para filtar e enviar arquivos tipo .doc e .docx

accept="application/msword" --- filtar e enviar arquivo do aplicativo word

accept="image/*,.pdf --- filtar e enviar arquivo do tipo imagem e tipo .pdf

accept=".jpg,.gif,.png" --- filtrar e enviar arquivo do tipo .jpg, .gif, .png

select

Tag de controle que fornece um menu de opções

name --- nome identificação do select

multiple --- atributo boolean que permite a seleção de mais de um item option.

size --- atributo que permite a quantidade de itens option a ser observado.

option --- é usado para criar um controle que representa um item dentro de uma tag select

option atributos

value --- ao inves de enviar o texto ao banco de dados, envia o value que pode ser "comprimido" para facilitar no server side e evitar um campo de dados grande, ou até mesmo o envio do código de um item.
